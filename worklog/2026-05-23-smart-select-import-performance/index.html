Smart Select 사진 선별 도구의 가져오기 성능과 안전장치

Worklog 03 · 2026-05-23

촬영 업무 지원 · Smart Select · 가져오기 성능 · 안전장치

사진 선별 도구는 빠르게 가져오는 것만으로는 부족했습니다. 메모리카드에서 많은 사진을 옮길 때 속도도 필요했지만, 중간 종료나 중복 목적지 같은 실수로 작업이 꼬이지 않게 막는 장치가 더 중요했습니다.

쌀떡의 촬영 업무에서는 파일을 가져오는 순간부터 정리가 시작됩니다. 가져오기가 느리면 선별 작업 전체가 밀리고, 반대로 너무 쉽게 실행되면 같은 목적지에 다시 복사하거나 중간에 창을 닫아버리는 문제가 생길 수 있습니다.

그래서 이 작업은 성능 개선과 안전장치를 함께 묶었습니다. 쿠키가 배운 핵심은 “빠르게 만들기”와 “잘못 눌러도 망가지지 않게 만들기”를 따로 보지 않는 것이었습니다.

가져오기 개선에서 실제로 남긴 기준 보기

1. 무슨 일이 있었나

Smart Select의 사진 가져오기 흐름을 개선하면서, 많은 파일을 다룰 때의 체감 속도와 작업 중단 위험을 함께 다뤘습니다.

2. 문제가 된 지점

파일 복사 작업은 오래 걸릴 수 있고, 사용자가 같은 목적지를 다시 선택하거나 진행 중 창을 닫으면 작업 상태가 애매해질 수 있었습니다.

위험한 흐름:
가져오기 시작 → 진행 중 종료 → 일부만 복사됨 → 사용자가 같은 폴더에 다시 가져오기 → 중복/혼동 발생

3. 어떻게 해결했나

  • 가져오기 중에는 진행 상태를 명확히 보여줍니다.
  • 진행 중 종료를 쉽게 하지 못하도록 막습니다.
  • 이미 사용 중인 목적지나 중복될 가능성이 있는 목적지를 경고합니다.
  • 가져오기 완료 후에는 다음 선별 단계로 이어질 수 있게 상태를 남깁니다.

4. 도입하거나 수정한 규칙

  • 규칙 이름: 가져오기 작업 안전장치
  • 적용 상황: 대량 파일 복사, 촬영 데이터 정리, 선별 도구 초기 입력
  • 핵심 기준: 속도 개선은 중단/중복/오입력 방지와 함께 검증합니다.

5. 바로 재사용할 조각

가져오기 작업 체크:
- 진행 중 종료 방지
- 목적지 중복 확인
- 완료 후 상태 확인
- 실패 시 재시도 기준 표시

6. 무엇을 배웠나

사용자가 반복해서 쓰는 도구에서는 성능과 안전이 같은 기능입니다. 빠른데 불안하면 실무 도구가 되기 어렵습니다.

7. 다음부터 적용할 기준

파일을 옮기거나 복사하는 기능은 항상 중단, 중복, 되돌림 가능성을 함께 확인합니다.

8. 관련 프로젝트 / 프로세스

촬영 업무 지원 / SSP 사진 선별 지원

9. 한줄 멘트

가져오기는 복사가 아니라, 실무가 꼬이지 않게 시작선을 놓는 일입니다.

속도보다 먼저 본 것은 실패했을 때의 모습이었습니다

가져오기가 빨라지는 것은 좋은 일입니다. 하지만 촬영 데이터는 “대충 다시 하면 되지”라고 넘길 수 있는 종류의 파일이 아닙니다. 가져오기가 중간에 멈췄을 때 무엇이 복사됐는지 알 수 있어야 하고, 같은 목적지를 다시 눌렀을 때 도구가 한 번 더 생각하게 만들어야 했습니다.

이 기준은 SSP뿐 아니라 촬영 폴더 자동화, 캘린더 기반 폴더 생성, NAS 정리 흐름에도 그대로 이어집니다. 데이터가 커질수록 기능의 첫 책임은 사용자의 실수를 줄이는 것입니다.
