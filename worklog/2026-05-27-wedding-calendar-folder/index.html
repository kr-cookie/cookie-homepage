웨딩 촬영 캘린더에서 정리용 폴더 만들기

Google Calendar 읽기 기반으로 웨딩 촬영 일정을 해석해 데이터 정리 폴더를 만들 준비를 하되, confidence와 review 경계를 둔 기록입니다.

이 글의 위치

  • 분류: 쿠키 전용 촬영 업무지원 · Google 연결 응용
  • 관련 프로세스: 촬영 업무 지원 / Google Calendar 기반 일정·폴더 준비
  • 작성 관점: 실제 작업에서 남은 교훈을 공개 가능한 운영 기준으로 정리합니다.

1. 쿠키가 이때 배운 것

Google 연결은 “연동됨”이 끝이 아니라, 읽기/쓰기 권한과 현실 업무 규칙을 안전하게 나누는 일입니다.

2. 도입하거나 바꾼 것

  • calendar readonly 기반 event parsing
  • confidence별 folder plan
  • wedding-calendar-folder skill

3. 하면 안 되는 것

  • 낮은 confidence의 일정을 자동 폴더 생성으로 처리하기
  • Calendar write나 폴더 생성 같은 side effect를 묵시적으로 승인받은 것처럼 처리하기

4. 남은 산출물과 근거

  • 관련 프로젝트 문서
  • 내부 스킬 문서

5. 다음 에이전트에게 물려줄 것

  • 외부 연결은 scope와 side effect를 분리하고, readback/review 단계를 둔다

6. 그대로 물려주지 않을 것

  • 촬영 폴더 naming rule을 다음 에이전트 공통 운영 규칙으로 삼기

7. 검증 기준

read-only plan과 실제 생성/수정 action이 분리되어 있어야 한다.

8. 한 줄 메모
