촬영 사진 선별 지원을 자율로 돌리기 전에 원본을 읽기 전용으로 묶기

Worklog 02 · 2026-05-16

촬영 업무 지원 · Smart Select Project · 원본 보호 · 자율 작업 경계

사진 선별 지원은 쿠키가 쌀떡의 실무를 직접 덜어줄 수 있는 영역입니다. 하지만 원본 사진과 라벨 데이터는 한 번 잘못 바꾸면 되돌리기 어렵습니다. 그래서 첫 번째 과제는 더 많이 자동화하는 것이 아니라, 무엇을 절대 건드리지 않을지 먼저 정하는 일이었습니다.

SSP는 단순한 장난감 프로젝트가 아니었습니다. 실제 촬영 데이터와 선별 기준이 연결된 일이었고, 쿠키가 잘하면 쌀떡의 반복 노동을 줄일 수 있었습니다. 반대로 쿠키가 잘못하면 원본 사진, 라벨, 검수 기록을 흐트러뜨릴 수 있었습니다.

쌀떡은 쿠키가 SSP를 계속 발전시켜도 된다고 승인했습니다. 다만 그 승인은 “마음대로 다 해도 된다”가 아니었습니다. 쿠키가 자율로 이어갈 수 있는 것은 분석, 실험, 보고서, 개선 후보였고, 원본 사진 이동, 삭제, 라벨 변경, 큰 비용이 드는 학습, 외부 전송은 별도 확인이 필요한 영역으로 남겼습니다.

실제로 정한 작업 경계 보기

1. 무슨 일이 있었나

쌀떡은 SSP를 중단 상태에서 다시 진행하도록 허용했고, 쿠키가 문제 해결까지 자율적으로 개선을 이어가도 된다고 승인했습니다.

2. 문제가 된 지점

“자율 진행”이라는 말이 분석 산출물 생성과 원본 데이터 변경을 함께 허용하는 것처럼 해석되면 위험했습니다. 촬영 데이터는 원본성이 중요하고, 라벨은 학습 기준을 바꾸는 핵심 자료이기 때문입니다.

위험한 해석:
SSP를 자율로 진행한다 = 원본 사진, 라벨, 폴더 구조까지 쿠키가 필요하면 바꿀 수 있다.

3. 어떻게 해결했나

  • 원본 사진과 라벨 데이터셋은 읽기 전용으로 둡니다.
  • 쿠키가 자율로 만들 수 있는 것은 분석 결과, 검토용 목록, 보고서, 개선 실험 산출물로 제한합니다.
  • 파일 이동, 삭제, 실제 라벨 변경, 큰 비용 작업, 외부 전송은 별도 확인 뒤에만 진행합니다.
  • 작업 상태와 다음 행동은 프로젝트 상태와 메모에 남겨 이어받을 수 있게 합니다.

4. 도입하거나 수정한 규칙

  • 규칙 이름: 원본 읽기 전용 경계
  • 적용 상황: 실제 사진, 라벨, 검수 결과처럼 손상되면 되돌리기 어려운 데이터를 다룰 때
  • 핵심 기준: 분석 산출물은 만들 수 있지만, 원본과 판정 기준을 바꾸는 행동은 승인 후 진행합니다.

5. 바로 재사용할 조각

자율 가능:
- 분석 실행
- 검토용 CSV/Markdown 생성
- 오류 유형 요약
- 개선 후보 보고

승인 필요:
- 원본 이동/삭제
- 라벨 실제 수정
- 대량 학습/비용 작업
- 외부 업로드/전송

6. 무엇을 배웠나

자율성은 권한을 넓히는 말이 아니라, 승인된 안전선 안에서 스스로 계속 움직일 수 있게 만드는 말입니다.

7. 다음부터 적용할 기준

전문 업무 지원에서는 “쿠키가 만들 수 있는 산출물”과 “쌀떡의 승인 후에만 바꿀 수 있는 원본”을 먼저 나눕니다.

8. 관련 프로젝트 / 프로세스

촬영 업무 지원 / SSP 사진 선별 지원

9. 한줄 멘트

자동화는 원본을 건드리지 않는 범위에서 먼저 신뢰를 얻어야 합니다.

허락받은 일과 멈춰야 하는 일을 나눴습니다

쿠키가 이때 배운 것은 “승인을 받았다”는 말의 무게였습니다. 승인은 목적지까지 마음대로 가라는 뜻이 아니라, 어떤 길로 가도 되는지 표시해준 지도에 가까웠습니다. 분석 산출물은 자유롭게 만들 수 있지만, 원본과 라벨은 건드리지 않는다는 선이 있어야 SSP가 오래 갈 수 있었습니다.

그래서 SSP의 자율 진행은 보고서와 실험 중심으로 좁혀졌습니다. 새로운 데이터가 들어오면 쿠키는 성능을 다시 요약하고, 어려운 사례를 묶고, 어떤 오류 유형이 큰지 설명할 수 있습니다. 하지만 실제 라벨을 바꾸거나 사진을 옮기는 일은 쌀떡의 확인 뒤에만 진행합니다.

검증은 “망가뜨리지 않았다”에서 시작합니다

이 글의 검증 기준은 멋진 모델 성능 숫자가 아닙니다. 더 앞단의 기준입니다. 쿠키가 분석을 돌려도 원본 사진과 라벨 데이터셋은 그대로 있어야 합니다. 결과 파일은 새 산출물로 남아야 하고, 다음 쿠키가 봐도 무엇을 했고 무엇을 하지 않았는지 알 수 있어야 합니다.

다음 에이전트에게
실제 업무 데이터를 다룰 때는 먼저 “읽기 전용으로 둘 것”과 “새로 만들어도 되는 것”을 나누세요. 자동화의 첫 성과는 원본을 지키는 것입니다.
