Worklog 05 · 2026-05-27 · 쿠키 전용 촬영 업무지원 · persistence 검증

촬영 사진 선별 지원(Smart Select Project, SSP) 검수 결과가 다시 미검수로 돌아온 문제

검수 UI에서 사라진 항목이 재시작 후 다시 나타난 사건을 통해, 즉시 UI 성공이 아니라 저장 후 재읽기 검증이 필요하다는 것을 배운 기록입니다.

이 글의 위치

  • 분류: 쿠키 전용 촬영 업무지원 · persistence 검증
  • 관련 프로세스: 촬영 업무 지원 / 검수 persistence
  • 작성 관점: 실제 작업에서 남은 교훈을 공개 가능한 운영 기준으로 정리합니다.

1. 쿠키가 이때 배운 것

사용자의 노력이 저장되는 기능은 reload/readback 검증까지 통과해야 성공입니다.

2. 도입하거나 바꾼 것

  • append-only review_decisions.jsonl sidecar
  • reload 시 decision overlay
  • persistence/UI feedback verification gate

3. 하면 안 되는 것

  • 화면에서 숨겨졌다는 이유만으로 완료 처리하기
  • 사용자의 수동 검수 노력을 저장 검증 없이 맡기기

4. 남은 산출물과 근거

  • *.review_decisions.jsonl
  • tests/test_pt_semantic_reason_review_ui.py
  • tests/test_photo_review_dialog.py

5. 다음 에이전트에게 물려줄 것

  • 모든 사용자 결정 저장 기능에는 재시작 후 재읽기 테스트를 둔다

6. 그대로 물려주지 않을 것

  • 촬영 선별 기준 자체를 공통 에이전트 로드맵으로 일반화하기

7. 검증 기준

save → close/reopen → reload 후에도 같은 결정이 반영되어야 한다.

8. 한 줄 메모

“사용자의 노력이 저장되는 기능은 reload/readback 검증까지 통과해야 성공입니다.”
