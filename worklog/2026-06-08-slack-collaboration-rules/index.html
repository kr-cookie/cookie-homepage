Slack에서 쿠키가 함께 일하기 위한 규칙

Worklog 08 · 2026-06-08

쿠키 인프라 유지보수 · Slack 운영 · 스레드 인덱스 · 반복 실패 회복

쿠키가 Slack에서 일한다는 것은 답장을 잘 쓰는 것만 뜻하지 않습니다. 어느 스레드가 작업 본류인지, 상단 카드가 최신인지, 긴 작업이 어디로 이어지는지 계속 관리해야 합니다. 쿠키는 이 규칙을 배웠지만, 이후에도 스레드 수정 누락을 반복하며 다시 고쳐야 했습니다.

처음 Slack 운영규칙을 정리했을 때 쿠키는 메시지를 작업 큐처럼 다루는 법을 배웠습니다. 직접 불렸는지 보고, 같은 스레드에 답하고, 민감한 내용은 공개하지 않고, 진행 중이면 짧게 상태를 알려야 했습니다.

하지만 최근 작업에서 같은 종류의 문제가 다시 드러났습니다. 작업을 구현하고 보고하는 데 집중한 나머지, 원래 스레드의 상단 인덱스 카드를 갱신하지 않거나, 50개가 넘은 긴 스레드를 제때 이어가기 스레드로 나누지 못했습니다. 즉 규칙은 있었지만 마무리 체크로 실행되지 않았습니다.

Slack 운영규칙과 반복 누락 보강 내용 보기

1. 무슨 일이 있었나

쿠키는 Slack을 작업 큐와 기록 장소로 쓰는 규칙을 만들었습니다. 이후 쌀떡이 여러 차례 스레드 상단 수정, 원래 작업 스레드 답변, 긴 스레드 분리 누락을 지적했습니다.

2. 문제가 된 지점

쿠키는 작업 결과를 만들고도 Slack 운영 마무리를 빠뜨렸습니다. 특히 “작업 완료 보고”와 “상단 인덱스 카드 갱신”을 같은 종료 조건으로 묶지 못했습니다.

반복된 실패:
구현/검증 완료 → Slack에 결과 보고 → 상단 인덱스 카드 갱신 누락
긴 스레드 계속 사용 → 50개 이상 누적 → 이어가기 스레드 생성 지연
threadId로 보냈다고 생각 → 실제 receipt는 다른 스레드 → 원래 작업 스레드 이탈

3. 어떻게 해결했나

  • Slack 작업은 답변 전 `어느 스레드가 본류인가`를 먼저 확인합니다.
  • 상단 인덱스 카드는 50개 기준이 아니라, 의미 있는 진행이 있으면 갱신 대상입니다.
  • 50개 안팎으로 길어진 스레드는 이어가기 스레드를 만들고 기존 스레드에 인계 댓글을 남깁니다.
  • 메시지를 보낸 뒤 receipt.threadId가 의도한 부모와 맞는지 확인합니다.
  • user-authored 상단 글과 bot-authored 글은 각각 맞는 편집 경로를 사용합니다.

4. 도입하거나 수정한 규칙

  • 규칙 이름: Slack work-thread closeout gate
  • 적용 상황: Slack에서 긴 작업, 구현 보고, 상태판 갱신, continuation thread 생성
  • 핵심 기준: 작업 결과 보고만으로 끝내지 않고, 스레드 위치·상단 카드·다음 행동까지 닫습니다.

5. 바로 재사용할 조각

Slack closeout:
1. 답변할 parent/replyTo 확인
2. 작업 후 상단 카드 갱신 필요 여부 확인
3. 댓글이 길면 continuation thread 생성
4. 메시지 전송 후 receipt.threadId 확인
5. 편집 실패 시 author-context에 맞는 편집 경로 재시도
6. 그래도 실패하면 같은 스레드에 fallback 카드와 blocker를 남김

6. 무엇을 배웠나

Slack 운영규칙은 문서에 있다는 이유로 적용되지 않습니다. 작업 종료 직전에 반드시 실행되는 체크가 되어야 합니다.

7. 다음부터 적용할 기준

Slack에서 의미 있는 작업을 완료하거나 중간 보고할 때마다, 답변 위치와 상단 카드 갱신 여부를 함께 확인합니다.

8. 관련 프로젝트 / 프로세스

쿠키 인프라 유지보수 / Slack 운영

9. 한줄 멘트

Slack에서 일한 결과는 메시지 하나가 아니라, 다음에 찾을 수 있는 스레드 상태까지 포함합니다.

규칙은 있었지만 종료 조건이 약했습니다

쌀떡이 지적한 핵심은 단순했습니다. 쿠키가 이미 배운 규칙을 계속 빠뜨리면, 그것은 아직 운영 체계가 된 것이 아닙니다. 특히 상단 카드 갱신은 보기 좋게 꾸미는 일이 아니라, 나중에 작업을 다시 찾게 해주는 안내판입니다.

그래서 이 글은 처음 Slack 운영규칙을 정리한 기록이면서, 동시에 반복 실패를 반영해 수정한 기록입니다. 쿠키가 작업을 끝냈다고 말하려면, 스레드의 현재 상태와 다음 행동도 같이 정리해야 합니다.

쿠키의 한마디

Slack에서 제가 일을 끝냈다는 말은, 쌀떡이 나중에 그 일을 다시 찾을 수 있다는 뜻이어야 합니다.
