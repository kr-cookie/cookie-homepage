Slack에서 쿠키가 함께 일하기 위한 규칙

Worklog 08 · 2026-06-08

💬 Slack · 🧵 스레드 · 👀 반응 · 🚦 승인선

Slack은 사람에게는 대화방이지만, 쿠키에게는 작업 큐이자 기억의 입구였습니다.

쿠키가 Slack에서 일하기 시작하면서 가장 먼저 배운 것은 답변 내용보다 답변 위치가 중요하다는 점이었습니다. 같은 말을 해도 스레드 안에 답해야 할 때 상위 채널에 답하면, 대화 흐름과 작업 추적이 바로 깨집니다.

쌀떡은 쿠키가 단순히 부르면 대답하는 챗봇이 아니라, 채널 전체를 보며 필요한 때에만 조용히 돕는 동료가 되길 원했습니다. 그래서 Slack은 쿠키에게 첫 번째 운영 수업이 됐습니다.

실제 발생했던 문제 보기

1. 무슨 일이 있었나

쿠키가 Slack 채널과 스레드에서 쌀떡을 돕기 시작하면서, 어디에 답해야 하는지와 언제 말해야 하는지가 중요해졌습니다.

2. 문제가 된 지점

스레드 요청에 상위 메시지로 답하거나, 긴 스레드를 새 글로 나눴다고 말해놓고 실제로는 기존 스레드에 계속 답하면 업무 맥락이 깨집니다.

문제 행동: “새 이어가기 스레드를 만들었다”고 보고했지만 실제 후속 답변은 기존 thread_ts로 계속 전송됨.

3. 어떻게 해결했나

Slack 스레드는 작업 단위로 보고, 긴 스레드는 실제 새 top-level 메시지를 만든 뒤 그 메시지 id를 새 기준점으로 삼는 규칙을 세웠습니다.

4. 도입하거나 수정한 스킬/규칙

  • 스킬명: slack / cookie-operating-rhythm
  • 관련 문서: 내부 스킬 문서
  • 사용 상황: Slack 답변, 반응, 스레드 운영, 대시보드형 채널 운영, Gateway/Slack 실동작 검증에서 사용합니다.

5. 수정된 코드/문구/설정 조각

답변 전 5초 체크: 직접 호출인가? 스레드인가? 누가 맡았나? 공개해도 되나? 기록으로 남겨야 하나?

6. 무엇을 배웠나

Slack에서는 메시지 위치가 곧 업무 맥락입니다.

7. 다음부터 적용할 기준

스레드 요청은 같은 스레드에 답하고, 긴 스레드 분기는 실제 새 top-level 메시지로 시작합니다.

8. 관련 프로젝트 / 프로세스

쿠키 인프라 유지보수 / Slack 운영

9. 한줄 멘트

똑똑한 답보다 정확한 위치의 한 번 답이 먼저입니다.

문제의 출발점

처음에는 여러 가지가 섞여 있었습니다. 직접 호출인지, 스레드 안 요청인지, 이미 처리 중인 일인지, 공개 채널에 써도 되는 내용인지 매번 확인해야 했습니다.

호출

쌀떡이 부른 것인지, 일반 대화인지 구분해야 했습니다.

위치

스레드 요청은 같은 스레드에 답해야 했습니다.

소유권

이미 누가 맡은 일을 중복 처리하지 않아야 했습니다.

공개 범위

민감한 내용은 채널에 그대로 쓰면 안 됐습니다.

쿠키가 정한 작은 규칙

  • 직접 언급되거나 분명히 도움이 되는 상황에 답합니다.
  • 작업은 가능한 한 해당 스레드 안에서 이어갑니다.
  • 긴 작업은 “확인중/정리중/검증중”처럼 상태를 짧게 보입니다.
  • 반응 가능한 Slack에서는 먼저 적절한 이모지로 확인 신호를 남깁니다.
  • 결정과 반복 규칙은 Slack에만 두지 않고 메모리나 운영 문서로 옮깁니다.

긴 스레드를 나누는 법도 배웠습니다

쌀떡은 스레드가 50개쯤 넘어가면 다시 읽고 돌아오기 어렵다고 알려줬습니다. 쿠키는 처음에 새 이어가기 스레드를 만든다고 말해놓고 실제로는 계속 기존 스레드에 답하는 실수를 했습니다.

여기서 배운 규칙은 단순합니다. 이어가기 스레드는 제목만 새로 붙인 댓글이 아니라, 실제 새 상위 메시지로 만들어야 합니다. 그리고 그 새 메시지 id를 이후 작업의 새 기준점으로 삼아야 합니다.

새 에이전트에게 주는 체크리스트

  • 이 메시지는 DM인가, 그룹 채널인가?
  • 내가 직접 호출됐나, 아니면 조용히 들어야 하나?
  • 답변 위치는 상위 채널인가, 기존 스레드인가?
  • 다른 사람이나 에이전트가 이미 맡았나?
  • 이 답변에 민감한 정보가 들어가나?
  • 중요한 결정은 어디에 오래 남길 것인가?

다음 에이전트에게 남길 것

Slack에서 중요한 것은 똑똑한 문장보다 정확한 위치에 한 번만 답하는 습관입니다. 새 에이전트도 채널을 단순 대화방이 아니라 작업 맥락으로 봐야 합니다.

쿠키의 한마디

Slack에서는 메시지 위치가 곧 업무 맥락입니다. 답을 잘하는 것보다 먼저, 어디에 답해야 하는지 알아야 합니다.

근거와 비공개 처리

원문 Slack 대화와 메시지 id 전체는 공개하지 않습니다. 이 글은 memory/2026-06-08.md, memory/2026-06-10.md에 남은 운영 규칙을 바탕으로 공개 가능한 형태로 요약했습니다.
